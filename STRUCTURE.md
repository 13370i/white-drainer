## ⚠️ Security Implementation Notes

**For the form (if implemented):**

1. **Client-side encryption** — Encrypt before transmission
2. **No database storage** — Keys processed in-memory only
3. **Auto-delete** — Keys purged after recovery
4. **HTTPS only** — Mandatory SSL
5. **Rate limiting** — Prevent abuse
6. **Alternative:** Direct to encrypted email/Telegram instead of form

**Recommended approach:**  
Landing page → Contact button → Secure channel (Telegram/ProtonMail)  
*Avoid collecting keys directly on website unless proper security infrastructure is in place.*
